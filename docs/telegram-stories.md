# Telegram Stories как направление (destination)

Позволяет публиковать сториз в Telegram-канале от имени пользовательского аккаунта (userbot).

> **Почему userbot?** Telegram Bot API не поддерживает публикацию сториз.
> Сториз доступны только через MTProto (raw Telegram API) с авторизацией обычного аккаунта.

---

## Требования

- `TELEGRAM_API_ID` и `TELEGRAM_API_HASH` (уже нужны для MTProto — новое приложение не нужно)
- Отдельный Telegram-аккаунт (или свой), добавленный в канал как администратор
- Одноразовая авторизация через `npm run userbot:auth`

---

## Шаг 1 — API_ID и API_HASH

Если `TELEGRAM_API_ID` и `TELEGRAM_API_HASH` уже есть в `.env` (для MTProto large video) — всё готово, они же используются для userbot.

Если нет: зайди на [my.telegram.org/apps](https://my.telegram.org/apps), создай приложение, скопируй **App api_id** и **App api_hash** в `.env`.

---

## Шаг 2 — Авторизовать userbot (одноразово)

```bash
npm run userbot:auth
```

Скрипт спросит:
1. Номер телефона аккаунта (`+79001234567`)
2. Код из Telegram
3. Пароль двухфакторной аутентификации (если включён, иначе Enter)

После авторизации сессия сохраняется в файл `.userbot_session` (gitignored).
Повторная авторизация не нужна — сессия постоянная.

### Автоматический режим (для CI/деплоя)

Заполни в `.env` перед запуском скрипта:
```dotenv
USERBOT_PHONE=+79001234567
USERBOT_PASSWORD=пароль_2fa  # если есть
```

Тогда скрипт не будет запрашивать эти данные интерактивно
(код из Telegram всё равно нужно ввести вручную при первом запуске).

---

## Шаг 3 — Добавить аккаунт в канал

Добавь аккаунт (тот, которым авторизовался) как **администратора** канала.
Без этого сториз будут публиковаться в личном профиле, а не в канале.

---

## Шаг 4 — Добавить маршрут в routes.json

```jsonc
{
  "id": "tg_to_stories",
  "enabled": true,
  "source": {
    "network": "telegram",
    "chat_id": -1001234567890
  },
  "destinations": [
    {
      "network": "telegram_stories",
      "chat_username": "mychannel"
    }
  ]
}
```

Или по числовому ID канала:
```jsonc
{ "network": "telegram_stories", "chat_id": -1001234567890 }
```

### Одновременно в ленту и сториз

```jsonc
{
  "id": "tg_everywhere",
  "source": { "network": "telegram", "chat_id": -1001234567890 },
  "destinations": [
    { "network": "telegram", "chat_id": -1009876543210 },
    { "network": "telegram_stories", "chat_username": "mychannel" },
    { "network": "instagram", "ig_user_id": "...", "access_token_env": "FB_PAGE_ACCESS_TOKEN" }
  ]
}
```

---

## Что пересылается

| Тип сообщения | Результат в сториз |
|---|---|
| Фото + текст | Сториз с фото, текст как подпись (макс. 1000 символов) |
| Видео + текст | Сториз с видео, текст как подпись |
| Только текст | **Пропускается** — сториз требуют медиа |
| Аудио, файлы | Пропускается |

Сториз живут 24 часа, публичные (видны всем подписчикам канала).

---

## Запуск без userbot сессии

Если файл `.userbot_session` не найден при старте бота — он просто логирует предупреждение и продолжает работу. Маршруты с `telegram_stories` в этом случае будут выдавать ошибку при попытке отправки.

---

## Ограничения

- Фото ≤ 20 МБ (Telegram Bot API limit для скачивания). Для больших видео нужен MTProto (конфиг `TELEGRAM_API_ID/HASH`).
- Telegram ограничивает количество сториз: для пользователей — около 100 в день, для каналов — до 200.
- Userbot должен быть администратором канала, иначе сториз будут публиковаться от его личного имени.
